* ModbusRTU.c
	1. 접점(디지털 입력)데이터 10ms 간격으로 수집하여 변경되었다면 SqliteDB에 업데이트 한다.
	2. TCP/IP 소켓 통신 프로그램의 데이터 요청에 대한 응답(ModbusRTU protocol)
	3. Redis 라이브러리를 사용한 pub/sub 인터페이스 내장됨 => 용도는 셋팅프로그램에서 접점을 클리어할 때 사용된다.
	4. 특정경로에 저장된 config 파일을 읽어서 운영한다.

* SerialToEthernet.c
	1. 시리얼 통신과 소켓 통신이 스레드로 분리되어 통신한다.
	2. 컨버터 역할을 한다.( 시리얼 to 이더넷 또는 이더넷 to 시리얼 )
	3. 소켓 통신은 설정에 따라 서버 모드나 클라이언트 모드로 통신한다. 
	4. 특정경로에 저장된 config 파일을 읽어서 운영한다. 

* SetupManager.c
	1. 윈도우 기반 셋팅 프로그램과 인터페이스 하는 역할을 한다.
	2. UDP 소켓 통신으로 데이터를 주고 받는다.
	3. 디바이스 환경정보를 읽고 쓰는 기능을 한다.
	4. 접점정보 초기화 명령을 수신한 경우는 Redis 라이브러리로 ModbusRTU.c 쪽을 초기화정보를 보낸다.


* DeviceStatusCheck.c
	1. 디바이스 외부 파란LED를 제어한다.
	2. 현재 운영중인 프로그램이 모두 정상구동 중이면 파란LED를 켜고 모두 실행되지 않았다면 끈다.
	3. wiringPi 라이브러리를 사용하였다.
